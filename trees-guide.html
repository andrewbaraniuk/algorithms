Збалансовані бінарні дерева

AVL · Balanced BST

Бінарне дерево пошуку (BST) — структура, де кожен вузол містить ключ, і всі ключі в лівому піддереві менші, а в правому — більші. Проблема простого BST: при послідовній вставці воно виродиться у список, і складність стає O(n).

AVL-дерево (Адельсон-Вельський і Ландіс, 1962) — перше самобалансуюче BST. Гарантує, що різниця висот лівого і правого піддерева (balance factor) не перевищує 1.

Червоно-чорне дерево

Red-Black Tree · 1972

Винайдене Рудольфом Байєром як "symmetric binary B-tree". Це самобалансуюче BST, де кожен вузол має колір — червоний або чорний. Менш строге ніж AVL (висота ≤ 2·log₂(n+1)), але потребує менше ротацій при вставці/видаленні.

// П'ять незмінних правил

  1. Кожен вузол — або RED, або BLACK
  2. Корінь завжди BLACK
  3. Всі листові NIL-вузли — BLACK
  4. Якщо вузол RED → обидва його нащадки BLACK (немає двох червоних поспіль)
  5. Для будь-якого вузла: всі шляхи до листя мають однакову чорну висоту

// Приклад дерева

// Вставка: три випадки фіксу

Випадок 1 — «Дядько червоний»

Перефарбувати батька і дядька в BLACK, дідуся в RED. Рухатись вгору.

Випадок 2 — «Дядько чорний, трикутник»

Ротація батька → перетворити в лінію (випадок 3).

Випадок 3 — «Дядько чорний, лінія»

Ротація дідуся + перефарбування батька ↔ дідуся.

AVL vs Red-Black

AVL — суворіший баланс, швидший пошук. RB — менше ротацій, швидша вставка/видалення.

Sift Up (вставка)

Додати в кінець масиву. Поки порушується heap property — міняти з батьком. O(log n).

Sift Down (видалення кореня)

Замінити корінь останнім елементом. Поки порушується — міняти з більшим нащадком. O(log n).

Heapify (побудова з масиву)

Sift-down для кожного внутрішнього вузла знизу вгору. O(n) — швидше за O(n log n) послідовних вставок.

Heap Sort

Побудувати max-heap → n разів витягти максимум. O(n log n), O(1) додаткової пам'яті.

Розширювані дерева (Splay Trees)

Splay Tree · 1985 · Sleator & Tarjan

Splay-дерево — самоналаштовуюче BST без явного зберігання балансу або кольору. При кожному доступі до вузла він переміщується в корінь через splay-операцію. Нещодавно використані елементи знаходяться швидше.

Ключова ідея: часта операція → дешевша амортизована вартість. Немає гарантій на окремі операції, але амортизована складність O(log n).

// Три типи splay-кроків

  • Zig — x є дитиною кореня → одна ротація
  • Zig-Zig — x і батько з одного боку (обидва ліві або обидва праві) → дві ротації батька, потім x
  • Zig-Zag — x і батько з різних боків (трикутник) → дві ротації x

// Демонстрація: splay вузла до кореня

Переваги

Немає додаткових полів (колір, висота). Добре для кешу з локальністю доступу. Проста реалізація.

Погані гарантії для окремих операцій. Read модифікує дерево — проблема для concurrent access.

Б-дерева (B-Trees)

B-Tree · Bayer & McCreight · 1972

Б-дерево порядку m — узагальнення BST, де кожен вузол може мати до m−1 ключів і до m нащадків. Розроблено для мінімізації дискових операцій: один вузол = один блок диска (зазвичай 4–16 KB).

На відміну від AVL чи Red-Black, де вузол має 1 ключ і 2 нащадки, Б-дерево «розширюється вшир» — дерево залишається дуже плоским, і кількість звернень до диска мінімальна.

// Структура вузла (порядок 4)

// Інваріанти Б-дерева порядку m

  1. Кожен вузол має ≤ m−1 ключів і ≤ m нащадків
  2. Кожен не-кореневий внутрішній вузол має ≥ ⌈m/2⌉ нащадків
  3. Корінь має ≥ 2 нащадки (якщо не листок)
  4. Всі листки на однаковій глибині (дерево завжди збалансоване!)
  5. Ключі всередині вузла відсортовані

// Варіанти

B+ Tree

Дані тільки в листках. Листки з'єднані в зв'язаний список → ефективне сканування діапазонів. Використовується у більшості СУБД (InnoDB, PostgreSQL).

B* Tree

Вузли заповнені мінімум на 2/3 (замість 1/2). Рідший спліт — краща утилізація пам'яті. Використовується у HFS (Apple).

2-3 Tree

Б-дерево порядку 3. Кожен вузол має 2 або 3 нащадки. Простіша реалізація, дидактично корисне.

LSM-Tree

Log-Structured Merge Tree — альтернатива для write-heavy навантаження. Використовується у RocksDB, Cassandra, LevelDB.

Перетворення Фур'є (DFT / FFT)

Fourier Transform · FFT · O(n log n)

Перетворення Фур'є розкладає сигнал (або будь-яку послідовність чисел) на суму синусоїд різних частот. В алгоритміці важливе як потужний інструмент для множення поліномів, обробки сигналів і стиснення даних.

// Дискретне перетворення Фур'є (DFT)

X[k] = Σn=0N-1 x[n] · e−i·2π·k·n/N
де k = 0..N-1 (частотні компоненти)
x[n] — вхідний сигнал (N точок)
e−i·2πkn/N = cos(2πkn/N) − i·sin(2πkn/N)

Пряме обчислення DFT: O(N²) — надто повільно для великих N.

// FFT — Fast Fourier Transform (Кулі-Тьюкі, 1965)

Алгоритм «поділяй і владарюй»: рекурсивно ділить DFT розміру N на дві DFT розміру N/2 — парних і непарних індексів.

X[k] = E[k] + WNk · O[k]
E[k] — DFT парних елементів
O[k] — DFT непарних елементів
WNk = e−i2πk/N — «кручений» множник (twiddle factor)
Складність: O(N log N) замість O(N²)

Множення поліномів

Замість O(n²) наївного множення: FFT → поелементне множення → IFFT = O(n log n). Основа швидкого множення великих чисел.

Стиснення (JPEG, MP3)

DCT (косинусне перетворення) — варіант FFT. Перетворює блок пікселів/семплів у частоти, відкидає незначущі → стиснення.

Конволюція

Операція f*g у часовій області = поелементне множення F·G у частотній. Використовується в CNN (deep learning), обробці зображень.

String matching

FFT-based string matching: знайти всі входження патерну у рядку за O(n log n) замість O(n·m).

Обходи дерева (Tree Traversal)

DFS · BFS · Pre/In/Post-order

Обхід — систематичне відвідування всіх вузлів дерева. Вибір порядку залежить від задачі: для сортування — inorder, для копіювання — preorder, для видалення — postorder, для пошуку за рівнем — BFS.

// Інтерактивна демонстрація

Результат:—

// Чотири основні обходи

Inorder · L → N → R

Для BST дає відсортовану послідовність. Результат: 1 2 3 4 5 6 7

Preorder · N → L → R

Корінь перший — зручний для серіалізації/копіювання дерева. Результат: 4 2 1 3 6 5 7

Postorder · L → R → N

Батько після нащадків — для видалення дерева, обчислення виразів. Результат: 1 3 2 5 7 6 4

BFS / Level-order

Рівень за рівнем через чергу (queue). Результат: 4 2 6 1 3 5 7. Знаходить найкоротший шлях у незважених графах.

// Реалізація (TypeScript)

// Inorder — рекурсивний function inorder<T>(node: TreeNode<T> | null): T[] { if (!node) return []; return [...inorder(node.left), node.val, ...inorder(node.right)]; } // BFS — ітеративний через чергу function bfs<T>(root: TreeNode<T> | null): T[] { if (!root) return []; const queue = [root], result: T[] = []; while (queue.length) { const node = queue.shift()!; result.push(node.val); if (node.left) queue.push(node.left); if (node.right) queue.push(node.right); } return result; }

// DFS: стек vs рекурсія

  • Рекурсія: елегантна, але може переповнити стек для глибоких дерев (stack overflow при глибині ~10k у JS)
  • Явний стек: ітеративна DFS — безпечна для великих дерев, контрольована пам'ять
  • Morris Traversal: inorder за O(1) пам'яті — тимчасово модифікує вказівники дерева (threading)
